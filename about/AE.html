Working with Terraform

Terraform was the primary tool I used for Infrastructure as Code (IaC) in my projects. I deployed infrastructures ranging from Kubernetes clusters on on-premise machines to full cloud environments in Azure and GCP. Leveraging Terraform workspaces for different environments, I streamlined deployments through both pipelines and manual execution as needed.

My responsibilities included deploying and managing Kubernetes clusters, virtual machines, Docker containers, storage containers, databases, Databricks workspaces, Keycloak instances, and configuring various dependencies and applications. I also managed Azure network rules, private links, and more.

All of this was accomplished with a single Terraform configuration, whether executed manually or automatically via pipelines. The setup only required a code editor, one or more Git repositories, and a storage solution for managing Terraform state.

Biggest lesson from Terraform? terraform force-unlock... or the usual tricks—break the lease in Azure blobs or check the table in databases.

Azure DevOps

Azure DevOps is my go-to CI/CD platform, offering a complete suite of tools in one place—tasks, pipelines, boards, and more. Setting up pipelines is a breeze with built-in templates and dependencies. You simply set up a virtual machine, connect it using a Service Principal (SP) or Managed Identity, install necessary dependencies (like Docker, if required), and then execute the pipeline. The platform provides full visibility into progress, results, and logs. You can easily integrate with registries such as JFrog or Azure Container Registries, and securely manage sensitive data by connecting to Azure Vaults—no need to manually handle keys or secrets.

My experience with Azure DevOps includes setting up pipelines for a range of tasks. I've created pipelines that securely retrieve keys from Azure Vaults, start Databricks clusters, and trigger their internal workflows. I've also set up pipelines to build Docker images for web applications and deploy them to Azure Web Apps. Additionally, I've automated Terraform deployments, managed secret updates in Azure Vault, and packaged Python functions for publishing to a JFrog package registry. Once the functions are packaged, I deploy them to a Function App, ensuring everything runs smoothly and securely.

GitLab

I have extensive experience using GitLab for CI/CD, and I'm confident enough to say I can automate just about anything with it—even making a lightbulb light up if needed.
I've built pipelines for deploying Terraform configurations, managing Kubernetes cluster deployments, and setting up umbrella Helm chart repositories.
I've also created pipelines for tasks like sending notifications, managing config maps, updating certificates, running Ansible playbooks, containerizing apps, and automating version updates and package tagging checks. In one project, I tailored pipelines to each developer's needs, ensuring custom behaviors for different users within an organization, created reusable helper functions, scripts, and jobs across repositories to streamline management and improve efficiency.
These multi-use tools make it easier to handle recurring tasks, ensuring consistency and reducing the need for redundant configurations across projects. Additionally, I've set up and managed GitLab registries, integrating them seamlessly into the GitLab ecosystem.

GitHub Actions
