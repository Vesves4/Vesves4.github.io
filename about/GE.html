Game Engines

Unity

Unity was my first deep dive into game development. I've experimented with C# scripting, physics systems, UI design, and game mechanics. I've worked on small projects, from basic platformers to procedural terrain generation. Unity's asset store and vast ecosystem make it a powerful engine, and I've enjoyed tinkering with its capabilities.

Godot

Godot is my preferred engine when I want something lightweight and open-source. I love working with GDScript, which feels like a mix of Python and game-specific scripting. I've used Godot to prototype 2D games, set up animations, create tile-based maps, and experiment with node-based scene structures.

While I don't have professional experience in game development, I enjoy exploring different engines as a hobby. It's a great way to learn programming patterns, optimize performance, and dive into creative problem-solving.
