Data management

PostgreSQL

I've worked with PostgreSQL for handling relational data in various projects. From setting up databases, managing schemas, writing queries, and optimizing performance to handling backups and replication, I've had hands-on experience with PostgreSQL both on-premise and in cloud environments.

Microsoft SQL Server & Azure SQL

I've used Microsoft SQL Server in enterprise environments, handling database administration, writing stored procedures, managing backups, and optimizing queries. On Azure SQL, I've worked with PaaS implementations, setting up managed databases, configuring scaling, implementing security rules, and integrating them with applications.

Databricks

One of my bigger projects involved Azure Databricks for ETL pipelines and data processing. I worked on running and managing Apache Spark clusters, optimizing workloads, and automating pipeline execution through CI/CD. Even though my primary role wasn't writing the Python-based ETL scripts, I contributed to debugging, optimizing jobs, and handling infrastructure-related aspects of Databricks.

Each of these tools plays a different role in data management, and I've had practical experience using them in real-world production environments.
