Enterprise DevOps

Managing large-scale GitHub Server

My most significant project involved managing a GitHub Enterprise Server (GHES) hosted in Azure. The setup included two replica sets, supporting over 1,100 repositories and more than 1,000 active developers spread across 10 organizations.

Regular maintenance was critical, requiring frequent updates and security patches. Since our GHES instance was lagging behind a major version, we had to carefully upgrade it in incremental stepsâ€”applying 2-3 minor version updates at a time to mitigate the risk of breaking changes. To minimize disruption, all updates were executed over the weekend, ensuring developers experienced no downtime during work hours.

Additionally, we undertook a large-scale migration of GHES to the cloud. This complex transition involved moving all repositories, developer accounts, and organizational structures while ensuring a seamless experience for users. Extensive planning and execution were key to making this migration smooth and efficient.
