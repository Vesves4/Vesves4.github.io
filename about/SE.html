Scripting Languages

I'll keep it simple—I have experience with JavaScript + React.js, Python, Bash, Batch, GDScript, HTML, CSS, and Markdown.

For React & JavaScript, I built a custom "DevOps Control Panel" for one of my projects. It allowed teams to monitor and manage VMs, Docker containers, dependencies, firewall rules, and more. The frontend was React, while the backend was a simple Express.js API executing machine commands based on user actions. It was integrated with Keycloak for authentication and deployed on an internal domain with restricted access.
For Python, I worked on ETL pipelines in Databricks, contributing to Python-based data workflows. While I wouldn't call myself an expert, I had to learn enough to contribute effectively.
For HTML & CSS, I built simple Nginx-based web apps and helped developers debug Kubernetes ingress-related redirect issues.
For Bash & Batch, they're everywhere. If you're working in DevOps, scripting is second nature—whether automating deployments or simplifying repetitive tasks.
For Markdown, I've used it extensively for documentation—embedding links, dropdowns, images, and videos. Sure, "I am the documentation" is a great motto, but knowing how to write proper docs is still a must.
For GDScript, it's purely a hobby. I enjoy exploring different areas of IT, and game development with Godot is one of them.
